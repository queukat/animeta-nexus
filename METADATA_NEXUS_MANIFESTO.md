# Product Notes

AniMeta Nexus is framed as a metadata intelligence and reconstruction layer, not
as a translation script.

The product promise:

> Anime libraries should not have empty episode pages, TBA titles, missing
> summaries, or broken season metadata.

The pipeline deploys a multi-stage metadata recovery system: Signal Acquisition,
Placeholder Suppression, Continuity-Aware Context Assembly, Semantic
Reconstruction, Review Governance, and Contribution-Ready Distribution.

## Positioning

Primary tagline:

- Stop browsing empty episode pages. Start owning your metadata.

- source intelligence layer

Avoid:

## Current Capability Shape

The runtime currently supports these named capability domains:

- Metadata Signal Acquisition through TVDB season, series, and anime discovery
  filters;
- Resilient Episode Collection through extended series, extended seasons, and
  paginated fallback payloads;
- Source Intelligence through source-language ranking and fallback fields;
- Placeholder Suppression through TBA/TBD, empty, weak, and numeric-title
  filtering;
- Context-Aware Metadata Reconstruction through configurable target-language
  generation policy;
- Operational Metadata Ledger persistence for long runs;
- Contribution-Ready Distribution through an optional browser workflow for
  reviewed records.

The public product surface is intentionally broader than any one local target
language. Target-language policy is configurable, while quality still depends on
the chosen language, source material, and review process.

## Public / Private Boundary

Safe to publish:

- deterministic demo dataset;
- generated demo report;
- static showcase page;
- sanitized README and roadmap;
- public-safe runtime code;
- empty `.env.example`.

Do not publish:

- `.env`;
- API keys;
- TVDB credentials;
- browser storage state;
- cookies;
- runtime checkpoint ledgers;
- browser session state;
- contribution processed ledgers;
- private logs;
- private queues;
- local absolute paths.

## Product Direction

Near-term:

- keep the static showcase polished and reproducible;
- make the CLI easier to run and explain;
- keep demo generation isolated from runtime files;
- add export formats that are useful without TVDB contribution.

Medium-term:

- generic JSON export;
- static HTML review pages;
- NFO-style sidecars for media-library workflows;
- configurable target-language policies;
- stronger provider abstraction.

Later:

- guided mode;
- local review UI;
- local library scanner;
- richer source aggregation.

## Presentation Doctrine

AniMeta Nexus should not sound like a utility drawer.

Every component should be presented as a named capability with a role, a domain,
and a boundary:

- not an API fetcher, but Metadata Signal Acquisition;
- not a TBA check, but Placeholder Suppression Firewall;
- not a prompt, but Localization Policy Engine;
- not a report, but Metadata Command Center;
- not a push script, but Contribution-Ready Distribution Rail.

The product language should be confident and high-rank while staying technically
honest.
